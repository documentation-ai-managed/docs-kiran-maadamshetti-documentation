## Overview

Set up your Kiran Maadamshetti Documentation space in minutes. You create an account, log in, build a new documentation space, and configure your first project. This guide walks you through each step with clear instructions and code examples.

<Callout kind="tip">
  Use your brand color `{`#3B82F6`}` in custom themes for a personalized look.
</Callout>

## Create Your Account

Follow these steps to sign up.

<Steps>
  <Step title="Visit the Sign-Up Page" icon="user-plus">
    Navigate to the [Kiran Maadamshetti Documentation homepage](https://kiranmaadamshetti.com/docs) and click **Sign Up**.
  </Step>
  <Step title="Enter Your Details" icon="edit-3">
    Provide your email, full name, and a secure password. Accept the terms of service.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for a verification email and click the link to activate your account.
  </Step>
</Steps>

## Log In to Your Account

Choose your preferred login method.

<Tabs>
  <Tab title="Email & Password" icon="mail">
    Enter your registered email and password on the login page.
  </Tab>
  <Tab title="Google" icon="apple">
    Click **Continue with Google** and select your Google account.
  </Tab>
  <Tab title="GitHub" icon="github">
    Click **Continue with GitHub** and authorize the app.
  </Tab>
</Tabs>

## Create a New Documentation Space

After logging in, set up your space.

<Steps>
  <Step title="Select New Space" icon="plus">
    From the dashboard, click **New Documentation Space**.
  </Step>
  <Step title="Name Your Space" icon="edit">
    Enter a name like `My Project Docs` and add a description.
  </Step>
  <Step title="Choose Template" icon="layout">
    Select the **Blank MDX** template for flexibility.
  </Step>
</Steps>

## Initial Project Setup

Connect your space to Git for version control.

<CodeGroup tabs="GitHub,Local">
  ```bash
  # Clone your space repository
  git clone https://github.com/yourusername/my-project-docs.git
  cd my-project-docs
  
  # Add your first file
  echo "# Welcome" > README.mdx
  git add .
  git commit -m "Initial commit"
  git push origin main
  ```
  ```bash
  # Initialize locally
  mkdir my-local-docs
  cd my-local-docs
  git init
  echo "# My Local Docs" > index.mdx
  
  # Link to remote (after creating on platform)
  git remote add origin https://github.com/yourusername/my-local-docs.git
  git push -u origin main
  ```
</CodeGroup>

<Expandable title="Troubleshooting Common Issues" default-open="false">
  **Account Verification Fails?**  
  Check spam folder or resend the email from settings.

  **Git Push Errors?**  
  Ensure you use HTTPS with personal access token:  
  ```bash
  git remote set-url origin https://{YOUR_TOKEN}@github.com/username/repo.git
  ```

  Contact support at support@kiranmaadamshetti.com for help.
</Expandable>
